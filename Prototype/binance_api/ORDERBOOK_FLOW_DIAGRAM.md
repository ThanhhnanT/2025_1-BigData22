## ğŸ” Key Design Patterns

### 1. **Snapshot + Incremental Updates**
- Fetch full snapshot first
- Apply incremental updates on top
- Ensures data consistency

### 2. **Event Buffering**
- Buffer events while waiting for snapshot
- Apply buffered events after snapshot
- Prevents data loss during initialization

### 3. **Rate Limiting**
- Limit Redis writes to 10/sec per symbol
- Balance between latency and performance
- Prevents Redis overload

### 4. **SortedDict for Order Book**
- Efficient insert/delete: O(log n)
- Fast top-level access: O(1)
- Maintains sorted order automatically

### 5. **Update ID Validation**
- Only apply updates with higher ID
- Prevents out-of-order processing
- Ensures data integrity
